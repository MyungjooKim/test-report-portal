# 자동화 결과 연동 — 자동화 엔지니어 요청사항

> 2026-07-23, TR Portal(tr.rgrg.im) test-run 연동 준비.
> 목표: Playwright 테스트 결과가 QA 수행 보드의 **자동 🤖 칸**에 자동 기입되고,
> 결과 대시보드(automation 컬럼·Fail 사유·Jira 내보내기)까지 흐르는 구조.

## 1. 테스트 코드 규칙 (지금부터 적용 부탁 — 전달 경로와 무관한 공통 전제)

### ① TC ID 태깅 — 가장 중요

테스트 제목 맨 앞에 대괄호로 TC Manager 의 TC ID 를 표기합니다.
**태그 없는 테스트는 보드에 매칭되지 않고 "미태그" 카운트로만 잡힙니다.**

```ts
test('[SC-TRD-ORDR-034] 수량 0 입력 시 에러 노출', ...)          // 단일
test('[SC-TRD-ORDR-036 | SC-TRD-ORDR-084] ...', ...)            // 한 테스트가 TC 여러 개 커버
test('[SC-TRD-ORHT-041 ~ SC-TRD-ORHT-052] ...', ...)            // 연속 범위 (041~052 전개)
```

### ② 거래소 식별

거래소별 실행 시 아래 중 하나로 거래소가 드러나야 합니다:
- Playwright project 이름 또는 리포트 폴더명에 `[Binance] trade-order` 형식 프리픽스 (현행 리포트가 이미 이 형식)
- (실시간 리포터 도입 후) 리포터 옵션 `exchange: 'Binance'` 지정 — 이 값이 우선

### ③ 대상 빌드 버전 스탬프

테스트 실행 시 환경변수로 빌드 버전을 주입해 주세요:

```bash
APP_VERSION=v1.2.4 npx playwright test
```

- 결과가 어느 빌드에서 나왔는지 보드에 기록됩니다 (버전 전환 시 ⚠ 재검 배지의 기준)
- 미지정 시 보드의 현재 대상 버전으로 폴백되지만, 명시를 권장

## 2. 결과 전달 경로

### 경로 A — 리포트 ZIP 업로드 (지금 바로 가능)

- HTML 리포트 폴더(임베드 `report.json` 포함, 현행 형식 그대로)를 **zip** 으로 압축
- tr.rgrg.im 결과형 프로젝트 → "결과 소스 업로드"로 업로드 (100MB 초과도 청크 업로드로 처리됨)
- 스크린샷·영상·트레이스 첨부까지 대시보드에서 딥링크로 열람 가능

### 경로 B — 실시간 리포터 (곧 제공 예정, 권장)

QA 팀이 단일 파일 리포터(`tr-run-reporter.js`)를 제공하면 `playwright.config.ts` 에 한 줄 추가:

```ts
reporter: [['list'], ['./tr-run-reporter.js', {
  base: 'https://tr.rgrg.im',
  runId: '<보드에서 복사>',
  token: '<보드에서 복사>',      // Google 로그인 불필요 — 보드별 업로드 토큰
  exchange: 'Binance',
}]],
```

- 테스트가 끝날 때마다(onTestEnd) 결과가 수행 보드의 자동 🤖 칸에 실시간 반영
- 네트워크 실패 시 버퍼 후 재시도 — 테스트 실행을 방해하지 않는 fire-and-forget
- CI/로컬 어디서든 동작 (세션 로그인 불필요, 토큰만)

## 3. 엔지니어 분들께 확인·요청드릴 것

| # | 항목 | 목적 |
|---|---|---|
| 1 | 최신 리포트 샘플 1건 (태깅 반영본) | 태그·거래소 파싱 검증 |
| 2 | 테스트 제목 태깅 현황 (미태그 비율) | 매칭률 예측, 태깅 백필 계획 |
| 3 | 실행 환경 (CI/로컬, tr.rgrg.im 으로 https 아웃바운드 가능 여부) | 경로 B(실시간) 성립 조건 |
| 4 | 거래소별 실행 단위 (Playwright project? 폴더 분리? 병렬?) | 리포터 exchange 옵션 설계 |
| 5 | 1회 실행 규모·소요 시간 (테스트 수, 재시도 정책) | 서버 수용량·폴링 주기 산정 |
| 6 | APP_VERSION 주입 가능 여부 (CI 변수 등) | 빌드 버전 스탬프 |

## 4. QA 팀(우리) 쪽 남은 작업 — R2

- `POST /api/runs/:id/auto-results` API (uploadToken 인증) — 보드 자동 칸 기입
- `tr-run-reporter.js` 단일 파일 제공 + 보드 화면에서 다운로드·설정 스니펫 복사 UI
- 화면 10초 폴링(변경 셀만 부분 패치) — 수행 중 실시간 반영

경로 A 는 오늘도 동작하므로, 태깅(①)만 맞춰지면 즉시 취합·Jira 내보내기가 가능합니다.
