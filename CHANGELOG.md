# Changelog

## [0.8.3] - 2026-07-16

### Changed — 결과 컬럼 라벨 다단 헤더 조합
- 상세 대시보드 결과 컬럼을 "Win/MAC-Tester 1", "MAC-Binance" 처럼 기기+테스터/거래소 조합으로 표시
  — 템플릿에 따라 기기가 헤더 행 위(병합) 또는 아래(서브 행) 어느 쪽에 있어도 인식
- 헤더 행의 빈 셀은 상위 행 값으로 보충 — 중요도 등 상위 행에만 있던 축 컬럼 인식 개선

## [0.8.2] - 2026-07-16

### Changed
- 상세 대시보드 축 기본 제외 목록 추가 (사용자 지정): 거래소·소분류·중분류·대분류·카테고리
  — 거래소는 결과 컬럼별 축과 중복, 분류 계열은 노이즈. data/settings.json 의 detail.excludeAxes 로 조정 가능

## [0.8.1] - 2026-07-16

### Fixed
- 상세 대시보드 접기/펼쳐보기 미동작 — 이 프로젝트의 `.hidden` 은 요소별 정의 방식인데
  `.dash-detail.hidden { display: none }` 규칙이 누락되어 캐럿만 바뀌고 내용이 남던 문제

## [0.8.0] - 2026-07-16

### Fixed — 헤더 행 자동 탐지 (신형 TC 템플릿 대응)
- 1행이 "{시트명}" 표지행인 템플릿에서 실제 헤더(TC ID 행, ~9행)를 자동 탐지 —
  Binance&BingX 리포트 기준 4건 → 8,000+건으로 정상화. 시트 중간 반복 헤더 행도 제거
- load 시점 정규화라 기존 업로드 리포트도 재업로드 없이 즉시 정상화 (통계·AI Q&A·커스텀 지표 공통)
- Win / MAC / Win/MAC 헤더를 결과 컬럼 별칭에 추가

### Added — 📊 상세 대시보드 (특징 축별 결과 분포)
- 특징 축(거래소·기기·테스터·우선순위·Smoke 등) 자동 감지 — 카디널리티 기반 값 프로파일,
  결과서마다 다른 헤더명을 그대로 라벨로 사용. 결과 컬럼 헤더는 기기·테스터 축으로 표시
- 발동 조건(축≥1 && 결과≥10건) 충족 시 기본 대시보드에 "상세 대시보드" 버튼 노출,
  축별 Pass/Fail/N-T/N-A 스택바 렌더링. 다중값 셀(HL, GT) 분해, 체크마크(∨/V/Y) 정규화
- 기준값은 data/settings.json 의 detail 섹션으로 재배포 없이 조정 가능
  (minAxes·minTotal·maxAxisCardinality 등, 기본값은 lib/detail-stats.js)

### Added — 🤖 자동화 결과 태그
- 업로드 팝업에 "자동화 결과" 체크박스 (파일·Sheets 공통), 리포트 목록에 🤖 자동화 뱃지

## [0.5.0] - 2026-07-15

### Added — 🔬 Fail 분석 및 결함 후보 (자동)
- Fail 이 있는 gsheet 리포트의 대시보드에 AI 결함 분석 섹션 고정 표시 (Fail 항목과 AI 질문 사이)
- 내용: 집중 영역 요약 → 동일 결함 의심 클러스터(TC 묶음+근거) → 수정 우선 대상

  안 바뀌면 저장본 즉시 표시(비용 0). Fail 0건이면 섹션 미표시. "다시 분석" 수동 버튼 제공

## [0.4.0] - 2026-07-15
