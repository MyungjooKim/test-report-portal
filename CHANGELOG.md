  tc-man 원본 에러 메시지(detail) 보존·표시
- 진단 기록: tc-man 미들웨어 예외 반영 확인(307→503), 남은 것은 tc-man `EXPORT_API_KEY` 설정뿐

## [0.19.3] - 2026-07-24

### Changed — TC Manager 스냅샷 선택 UI: 접이식 트리 + 최신순
- 단일 `<select>` → 프로젝트별 **접이식 트리**. 프로젝트는 접기/펼치기(개수 배지),
  프로젝트 안 버전은 **최신순** 정렬 + 최근 5개(`TCMAN_RECENT_N`)만 노출 + "이전 버전 N개 더보기"
- 각 프로젝트 최신 버전에 **"최신" 배지** — 대상 버전 즉시 식별
- **검색은 보조** — 평소엔 트리, 못 찾을 때만. 검색 중엔 접기·더보기 무시하고 매칭 전부 펼침
- 긴 Jira epic/릴리즈 버전명(`EPIC-1234 / v3.1.2-rc`) 대응 — 버전 강조 + 설명·메타 2줄, 카드형
- 배경: 프로젝트(플랫폼)는 적지만 테스트 주기가 빨라지며 스냅샷 버전이 시계열로 깊게 쌓이는 구조 대비
- 선택 시 보드 이름·스냅샷·거래소 축 자동 채움 동작 유지. 접근성(radio·키보드·role) 유지
- **정렬은 createdAt(생성 시각) 최신순으로 고정** — id 는 UUID(무작위), version 은 사람이 지은
  자유 문자열(v10<v2 함정·"EPIC-.../v3.1" 파싱 불가)이라 정렬 근거로 신뢰 불가. 방어: createdAt
  무효/누락 스냅샷은 항상 목록 최하위, 카드에 생성일 명시(🕐)로 순서 근거를 UI 에 노출
- `docs/TCMAN_EXPORT_API_ISSUE.md` 에 향후 논의 후보 기록 — 스냅샷 성격 플래그(released/test),
  서버 페이징·검색, createdAt 형식 보장 (연동 성공 후 목록이 커지면 검토)

## [0.19.0] - 2026-07-23

### Added — test-run R2: Playwright 리포터 실시간 자동화 기입
- **tr-run-reporter.js** (단일 파일, 의존성 없음) — playwright.config 한 줄 + 환경변수(TR_BASE/TR_RUN_ID/
  TR_TOKEN/TR_EXCHANGE/APP_VERSION)로 설치. onTestEnd 마다 3초 배치 push, 실패 버퍼 재시도,
  종료 시 전송/미태그 요약. 테스트 실행을 방해하지 않는 fire-and-forget
- **POST /api/runs/:id/auto-results** — 보드 uploadToken 인증(Google 세션 무관, CI/로컬 어디서든).
  TC ID 태그(단일/멀티/범위)·거래소([Binance] project 명) 파싱은 기존 어댑터 규칙 재사용.
  재시도 통과(flaky)는 최종 Pass + 시도 이력 스레드 보존, 대상 거래소 외 셀은 기록 제외
- **보드 "🤖 자동화 연동" 모달** — 리포터 다운로드, 실행 값 복사(env 스니펫), 토큰 재발급
- **10초 증분 폴링** — 변경 행만 부분 패치 + 조용한 하이라이트, 편집 중 셀 갱신 보류(다음 폴링 재적용),
  백그라운드 탭 쉼. 침입 방지 4원칙 준수
- 미태그 자동화 테스트 카운트 칩(최근 제목 툴팁) + 자동 칸/스레드에 에러 첫 줄(detail) 표시
- 버전 폴백: 리포터 APP_VERSION → 보드 대상 버전 — 다른 버전 유입은 스탬프 유지 + ⚠ 재검 배지

## [0.18.0] - 2026-07-23

### Added — Suite(시트) 접이식 섹션 + 3-소스 발행 (대량 TC·소스 컬럼 대응)
- 보드 그리드 3단 구조: **Suite(시트) 섹션 > 대분류›중분류 그룹 > TC 행** — 섹션 헤더에
  진행률·Fail·TC 수 요약, 클릭으로 접기/펼치기. 200 TC 초과 보드는 기본 접힘, 시트 1개면 현행 유지
- 발행을 **3-소스로 확장**: 자동 칸 → automation, 수동 칸 → manual(기입분만), 최종 칸 → test-run(전량)
  — 결과 대시보드에 기존 취합과 같은 소스 컬럼·소스별 축·커버리지가 나타남
- 취합 최종에 **권위 규칙**: test-run 소스가 있으면 그 값이 최종 — 보드의 커버리지 인지 파생
  (부분 커버리지 자동 Pass 단독 = N/T 등)을 D 규칙이 덮어쓰지 않음. 타 소스 결정값이 어긋나면
  불일치 표시는 유지. 기존(비 test-run) 취합은 무회귀
- 재발행만 하면 기존 보드도 3-소스 형식으로 갱신됨
